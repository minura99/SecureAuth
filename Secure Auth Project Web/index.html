SecureAuth - Behavioral Biometrics for Enhanced Authentication Systems

Get Research Paper Watch Demo

Liter​ature Survey

Behavioral biometrics leverage user behavior for authentication—using gait, typing patterns, mouse movement, and voice. These modalities are hard to replicate and ideal for continuous, passive authentication. Gait and voice were modeled with CNNs, while typing and mouse behaviors used RNNs. This system enhances traditional security methods, offering real-time, hardware-agnostic, privacy-conscious authentication.

Research Gap

Multi-modal behavioral biometric systems, integrating gait, keystroke, mouse, and voice data, offer strong security potential but face key challenges. These include variability in user behavior, synchronization of different data types, high computational demands, and privacy concerns. Systems must also adapt to behavioral changes over time, scale to diverse user groups, and remain user-friendly and cost-efficient. Addressing these issues is essential for practical, secure, and widespread adoption.

Research Problem

  • Difficulty replicating behavioral patterns
  • Integrating diverse data types
  • Real-time authentication challenges
  • Balancing security and usability

Objectives

To develop a multi-modal authentication system that integrates gait analysis, keystroke dynamics, mouse movements, and voice authentication into a unified, cohesive solution. The system aims to provide continuous, secure, and efficient user verification, addressing weaknesses in traditional and single-modality biometrics.

  • Collect and preprocess data from publicly available behavioral biometric datasets.
  • Develop deep learning models: CNNs for gait and voice; LSTMs for keystroke dynamics; CNNs for mouse dynamics.
  • Extract robust features and apply cosine similarity to generate similarity scores for each modality.
  • Fuse the outputs using a weighted late fusion strategy.
  • Ensure real-time operation, cross-platform compatibility, and user privacy by applying anonymized local processing and encryption techniques.

Methodology

The methodology involves collecting behavioral data, training CNNs and RNNs on different modalities, and using cosine similarity with weighted fusion for identity verification. Public datasets were used and privacy was ensured via anonymized, local data processing.

    • Collect and preprocess data from gait, keystroke dynamics, mouse movements, and voice.
    • Train CNNs for gait and voice recognition; use LSTMs for keystroke and CNNs for mouse movements.
    • Compute similarity scores using cosine similarity for each modality.
    • Fuse results using a weighted late fusion strategy for robust authentication.
    • Evaluate system performance with metrics such as FAR, FRR, precision, recall, and AUC.
